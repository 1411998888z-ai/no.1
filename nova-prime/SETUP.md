# STEP 0 アンケート｜運用セットアップ

契約者に回答してもらうための、配布〜収集の仕組み。構成は次のとおり（すべて無料）。

```
契約者 ─(手動でURL送付)→ アンケートページ ─(提出)→ Googleスプレッドシート
        GitHub Pages で公開           GAS が受信して1行追記        講師が閲覧
```

- ページ配信：**GitHub Pages**（このリポジトリ）
- 回答の収集：**Google スプレッドシート + GAS**
- 本人特定：URLの `?name=…&mentor=…&cid=…`（ログイン不要）
- 配布：運営が **admin.html でリンクを作り、手動で送信**

---

## セットアップ（初回だけ・15分）

### ① スプレッドシートと受信スクリプト（GAS）

1. Googleドライブで**スプレッドシートを新規作成**（例：「STEP0回答」）。
2. URLの `/d/` と `/edit` の間の文字列（＝スプレッドシートID）をコピー。
3. スプレッドシートの **拡張機能 → Apps Script** を開く。
4. `nova-prime/apps-script/Code.gs` の中身を全部貼り付け、`SHEET_ID` に②のIDを入れて保存。
   - 提出のたびにメール通知が欲しければ `NOTIFY_EMAIL` にアドレスを入れる（任意）。
5. **デプロイ → 新しいデプロイ → 種類「ウェブアプリ」**
   - 実行するユーザー：自分
   - アクセスできるユーザー：**全員**
6. 発行された **ウェブアプリURL（末尾 `/exec`）** をコピー。

### ② ページに受信URLを接続

`nova-prime/index.html` の先頭付近：

```js
const ENDPOINT = "";   // ← ①-6 の /exec URL をここに貼る
```

貼ってコミットすれば、提出がスプレッドシートに届くようになる。
（空のままだと「テストモード」＝送信せず控え出力のみ。）

### ③ GitHub Pages で公開

1. GitHub のリポジトリ **Settings → Pages**。
2. Source を「Deploy from a branch」、Branch を公開用ブランチ（例 `main`）/ `root` に設定。
3. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/nova-prime/` で公開される。

> 注意：`admin.html`（運営用リンク発行ツール）も同じ場所に置かれます。URLを知る人だけが使える運用にしてください（契約者には配らない）。

## 毎月の運用（契約者が増えても手間は増えない）

新規契約が出るたびに、運営がやることは**2ステップだけ**：

1. **リンクを作る** … `admin.html` を開き、氏名・担当講師を入力 → 個別リンクが生成される。
2. **送る** … 生成された文例ごとコピーして、LINE / メール等で本人に送る。

契約者が10人でも100人でも、ページ側は無変更。回答はすべて同じスプレッドシートに1行ずつ集まり、`氏名`・`担当講師`・`提出日時`の列で誰の分か判別できます。担当講師は自分の名前で絞り込めば、面談前に自分の担当分だけを読めます。

### 配布タイミング（心理設計上の推奨）

- **契約直後24時間以内**に送る（クーオフ期間の“空白”を作らないため）。
- **3〜4日たっても未提出**なら1回だけリマインド。
- 提出後に面談（STEP 1）を設定。

## ファイル

| ファイル | 役割 |
| --- | --- |
| `index.html` | 契約者が回答するアンケート本体（公開） |
| `admin.html` | 運営がリンクを発行するツール（公開場所に置くがURLは非共有） |
| `apps-script/Code.gs` | 回答をスプレッドシートに書き込むGAS |
| `artifact.html` | claude.ai プレビュー用の派生版（運用では未使用） |

## よくある調整

- **設問の追加・変更**：`index.html` の `STEPS` を編集し、`Code.gs` の `COLUMNS` に対応する行を足す。
- **提出のメール通知**：`Code.gs` の `NOTIFY_EMAIL` を設定。
- **LINE等での自動送信に切り替えたくなったら**：`admin.html` の手動発行を、契約者マスタ＋自動送信に置き換える拡張が可能（別途相談）。
