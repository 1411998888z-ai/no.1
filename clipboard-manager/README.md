# 📋 Copypad — コピペリスト管理ツール

コピペしたいテキストを **ファイル（フォルダ）別に管理** し、**ワンタッチでコピー** できるツールサイトです。
サーバー不要・完全静的。データはお使いのブラウザ内（localStorage）に保存されます。

## 特長

- 🗂️ **ファイル別管理** — 用途ごとにファイルを分けて整理（作成・名前変更・削除）
- 📋 **ワンタッチコピー** — カードのボタンを押すだけでクリップボードにコピー
- 🔍 **検索** — ラベル・本文をまたいで絞り込み（`Ctrl/⌘ + K` でフォーカス）
- 💾 **自動保存** — 入力内容はブラウザに自動保存（次回開いてもそのまま）
- ⬇️⬆️ **エクスポート / インポート** — 全データを JSON で書き出し・読み込み（端末間の移行やバックアップに）
- 🌓 **ライト / ダーク** — OS設定に自動追従＋手動切替
- 📱 **レスポンシブ** — スマホでもサイドバー開閉で快適に操作

## 使い方

1. `index.html` をブラウザで開くだけで動きます。
2. 左の「＋」でファイルを追加 → 右上「＋ 項目を追加」でコピペ内容を登録。
3. カードの「📋 コピー」でクリップボードにコピー。

> 本文が長い場合はカードのテキスト部分をクリックすると全文を展開できます。

### ショートカット

| キー | 動作 |
| --- | --- |
| `Ctrl / ⌘ + K` | 検索にフォーカス |
| `Ctrl / ⌘ + Enter` | 編集モーダルで保存 |
| `Esc` | モーダル / サイドバーを閉じる |

## 公開する（GitHub Pages）

このフォルダをそのまま配信すれば動作します。GitHub Pages で公開する場合は、
リポジトリの Settings → Pages でブランチを選び、対象ディレクトリを
`clipboard-manager/` にすればアクセスできます（もしくはこのフォルダの中身を
Pages 用ブランチのルートに配置）。

## データについて

- 保存先は各ブラウザの `localStorage`（キー: `copypad.v1`）です。**端末・ブラウザをまたいでは共有されません。**
- 別端末へ移したいときは「エクスポート」で JSON を保存し、移行先で「インポート」してください。
- ブラウザのデータを消去すると内容も消えるため、大切なデータは定期的にエクスポートを推奨します。

## 構成

```
clipboard-manager/
├─ index.html      # 画面
├─ css/style.css   # スタイル（デザイントークン / レスポンシブ / ダークモード）
└─ js/app.js       # ロジック（状態管理・保存・コピー・入出力）
```

依存ライブラリなし（バニラ HTML/CSS/JS）。
